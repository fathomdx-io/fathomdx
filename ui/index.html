How this works
  1. The command installs the Fathom helper on that machine (via npx fathom-agent).
  2. It redeems this pair code to get a long-lived API key, written to ~/.fathom/agent.json.
  3. Then it registers the helper as a system service (systemd on Linux, launchd on Mac, a scheduled task on Windows) and starts it.
  4. The helper starts heartbeating; this dashboard detects it and closes this dialog.
  5. Need to run on a different machine later? Re-open this dialog, get a new code. Re-pairing rotates the key automatically.

What this does
  1. npx -y fathom-agent@latest fetches the newest release from npm, bypassing any cached older copy.
  2. install rewrites the systemd/launchd/Windows service file to point at the new binary path, then restarts it.
  3. If the helper isn't running as a service, substitute run for install to just start it in the foreground.
  4. The helper reads the same ~/.fathom/agent.json, so you don't need to re-pair.
